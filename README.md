# The Lion Landscaping Website Workflow

## Project Overview

Create a responsive website for **The Lion Landscaping**, a landscaping business serving the **Tampa Bay area**.

The website consists of four HTML pages that share a common design and styling.

## Development Workflow

### Phase 1 – Home Page

* Create `index.html`
* Build the shared header/navigation
* Create hero section
* Add company introduction
* Add featured services preview
* Add call-to-action section
* Add footer

---

### Phase 2 – Global Styling

Create `style.css` containing:

* Color palette
* Typography
* Navigation styling
* Buttons
* Hero styling
* Responsive layout
* Shared utility classes
* Footer styling
* Mobile responsive breakpoints

All pages should use the same stylesheet.

---

### Phase 3 – JavaScript

Create `main.js` containing:

* Mobile navigation toggle
* Smooth scrolling (if applicable)
* Active navigation highlighting
* Future reusable JavaScript functionality

Keep JavaScript lightweight and dependency-free.

---

### Phase 4 – Services Page

Create `services.html`.

Include:

* Shared header
* Services hero/banner
* List of landscaping services
* Service cards
* Call-to-action
* Shared footer

---

### Phase 5 – My Work Page

Create `mywork.html`.

Include:

* Shared header
* Portfolio/gallery section
* Project descriptions
* Responsive image grid
* Call-to-action
* Shared footer

Image placeholders may be used until real project photos are available.

---

### Phase 6 – Contact Page

Create `contact.html`.

Include:

* Shared header
* Contact information
* Contact form
* Service area information
* Google Maps placeholder
* Shared footer

The contact form does not require backend functionality.

---

## Design Guidelines

* Modern landscaping aesthetic
* Green and earth-tone color palette
* Clean typography
* Mobile-first responsive layout
* Consistent spacing
* Simple animations
* Fast loading
* Semantic HTML5
* Accessible markup

---

## Navigation

Every page should contain the same navigation.

* Home
* Services
* My Work
* Contact

The active page should be visually highlighted.

---

## Code Standards

* Use semantic HTML.
* Use external CSS only (`style.css`).
* Use external JavaScript only (`main.js`).
* Keep code clean and well commented.
* Avoid external frameworks.
* Use vanilla HTML, CSS, and JavaScript only.

---

## AI Generation Rules

When generating or updating files:

1. Always generate the **entire file**, never partial snippets.
2. Preserve the existing project structure.
3. Reuse the shared navigation and footer across every page.
4. Keep styling centralized in `style.css`.
5. Keep functionality centralized in `main.js`.
6. Maintain responsive design.
7. Do not include hosting or deployment instructions.
8. Ensure all internal links point to the correct HTML files.
9. Do not rename files unless explicitly instructed.
10. Maintain a consistent visual design across the entire website.
